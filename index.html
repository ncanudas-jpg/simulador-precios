Cómo funciona el simulador

Modelo de elasticidad

Se estima β para cada SKU usando regresión log-log sobre ventanas semestrales deslizantes (6 meses, deslizando 1 mes a la vez):

log(unidades + 1) = α + β × log(precio_real)

Fórmulas del simulador (las que pidió el profe)

unidades_simuladas = unidades_actuales × (precio_nuevo / precio_actual) ^ β
ingreso_simulado = precio_nuevo × unidades_simuladas
margen_simulado = (precio_nuevo − costo_unitario) × unidades_simuladas

Clasificación de SKUs

ELÁSTICO ★β < -1 y p < 0.10 — el cliente reacciona al precio
Elástico n.s.β < -1 pero p ≥ 0.10 — aparente elasticidad ruidosa
Inelástico-1 < β < 0 — cliente responde poco al precio
Sin reacciónβ ≥ 0 — sospechoso, requiere diagnóstico individual

Recomendación de acción

BAJARElástico con precio actual > precio óptimo p* = c × β/(β+1)
SUBIRElástico con precio actual < precio óptimo (subprecio)
SUBIR (direccional)Inelástico — hipótesis a validar con prueba A/B
MANTENERSin evidencia para mover el precio
INVESTIGARβ positivo sospechoso — revisar datos
Advertencia obligatoria del profesor: si el costo tiene inconsistencias, el margen es exploratorio, no definitivo. La base de costos de OfficeMax presenta inconsistencias documentadas (productos con precio < costo), por lo que los márgenes simulados son direccionales, no estimaciones absolutas. Validar con prueba A/B en 2-3 tiendas antes de rollout.
